Avalanche Insights ‘human-in-the-loop’ analysis tool redesign

Complete end-to-end overhaul of a complex product flow enabling users to extract nuanced insights from open-ended text data

Overview

Avalanche Insights, a consultancy specializing in progressive politics, sought to scale their services by developing a product that could extract key themes and insights from open-ended text data. They needed a redesigned user flow that could allow non-expert users to easily navigate the complex human-in-the-loop machine learning process. With this goal in mind, I was recruited to facilitate a design sprint and create a product that would deliver a valuable data exploration and insights experience for users.


Goals

- Redesign user experience for complex 'human-in-the-loop' machine learning text analysis
- Create an intuitive experience for a non-expert user


Role

- Design Sprint Facilitator
- UI/UX Designer


Tools

Research and Discovery

The project followed the design sprint framework, with a timeframe of 2-3 weeks due to additional commitments across the team. In the understand phase, we conducted a UX audit of the incumbent solution and user interviews to identify the biggest barriers to users achieving their goals. Many challenges were identified, including a disconnect between user expectations and actual application behavior, an inflexible flow limiting user actions to a linear process, and users unable to access the information they required in the interface.

To ensure solutions were grounded in feasibility, we conducted workshops with our data science lead, engineering manager, and Avalanche Insights analysts. This allowed for creative thinking and sourced ideas for utilizing proprietary data science. Through this process, we identified opportunities to meet user needs and developed a problem statement: How can we use semantic diversity cluster analysis, natural language processing, and classification models to create an accessible, usable, and feasible product flow that allows non-expert users to extract themes from open-ended text data?

Journey Map

Extensive discovery interviews allowed me to develop a journey map with a number of stages that were constantly discussed by users. Initially (collect) the users were excited to start a new product, and reported a general contentment with their survey fielding software.

The very next stage (analyse) provided the biggest frustration, this was the realisation - often described as overwhelming - that the user would be required to sift through all survey responses to tease out themes and insights. It was typically reported that, even though the survey fielding tools included features to address this stage, their functionality was insufficient for the user requirements.

The third stage (curate) included pulling together visualisations, key quotes, and highlights to create presentation decks to report back to the organisation. Although this didn't cause the same dread as the previous stage, there were still some challenges around exploring data visualisations and crafting relevant stories.

The final stage for the user (share) is the final handoff of the recommendations and findings. At this stage the user reported feeling relieved having come through the previous two stages.


Design and Prototyping

Sketching was conducted asynchronously, and Crazy 8’s method was used for divergent thinking. We held a session to share Crazy 8’s ideas and then asked each participant to work on a solution sketch, an end-to-end proposal to address the problem statement. A decider was enlisted to review the solution sketches and advise on the most promising idea to move into the testing phase.

My proposal was selected, and I gradually increased the fidelity from sketch to wireframe, then higher fidelity screens, and finally an interactive prototype for testing. I collaborated closely with the Product Manager and drew on our shared knowledge of non-expert users gained over the course of numerous discovery interviews. Each data science concept was separated in the product flow, allowing for prioritization of optimization if it proved valuable to the user.


Screens

Given the requirements set at the outset - broadly, use this data science approach and include these micro-interactions - it became very apparent that there would be real value in conducting a rumble whereby my proposal would be tested directly against the existing solution, following the same interview script, tasks, and success metrics.

A number of key features are presented in the following user flows:

Start Analysis

After the user has completed their upload to the platform, they can access the analysis dashboard in an empty state. There is a call to action, leading the user to begin the machine assisted analysis process.

Cluster analysis is performed on the upload and the user is able to start their analysis with some smart themes which are fully customisable. This feature was a key value proposition discovered through user interviews where users would describe feeling overwhelmed starting with a blank slate and would like a headstart on their analysis.

In the incumbent solution, the user was confused by the requirement to confirm every AI suggestion. My solution automatically adds the AI suggestions, while making it really straightforward to manage the suggestions.



Explore Smart Themes

The user can access everything in a single interface. They can explore themes and responses; search, sort, and filter; and generally move in the direction the analysis takes them. In the UX audit of the incumbent solution, users noted how restricted they were in the direction they could take with their analysis.



Add a Response to a Theme

Associating responses with themes is the defining micro-interaction required to complete the analysis. The interaction in my solution is based on familiar industry patterns often seen in email inbox management. Clicking a response to select, the user is presented with an action menu. The user can also drag responses onto a theme.



Add Multiple Responses to a New Theme

The user can select multiple responses and access the same action menu as above - or drag the responses on to an existing theme, or the 'Create new theme' button.

The UX audit surfaced some very clumsy interactions to add multiple responses to a theme, including holding down the shift key to multi-select. Although this action can be intuitive to a product designer, target users were not able to complete the task with this interaction.



See Similar Responses

See similar responses was a novel use of the existing data science which I developed during the ideation process. All responses are plotted on a vector map during the cluster analysis, see similar allows the user to select a reference point and explore the closest responses from the vector map. In allowing the user to surface and analyse multiple similar responses, the efficiency of their process is massively increased.



Organise Themes

Almost all users identified a requirement to organise themes and create theme hierarchies. During the UX audit this feature was identified as a major weakness for the incumbent solution, as the user was either fully or partly restricted in when they could complete this step - either at the end of the process or by navigating to a new tab.

My solution embedded the ability to organise themes at any point in the analysis, and in the interface that the user is working. Crucially, I understood that organising themes is an iterative process, theme hierarchies are fluid and will change as the analysis proceeds - making access a key requirement for the user.



Train Theme

Training a theme allows the user to provide direct feedback to the AI classification model. This is a key feature of the platform as it enables auto tagging, which can increase the efficiency of the analysis exponentially.

The incumbent solution delivered this interaction in a very busy interface, which caused confusion during user interviews. My solution utilises the thumbs-up, thumbs-down interaction (which users liked) but reduces the complexity of the interface and thus reduces cognitive load for the user. Within the drawer in my interface, the user can also add or remove responses which have been added by the user or added by AI - this interaction provides further feedback to the classification model without encroaching on the user's analysis flow.



Analysis Complete

The user has the autonomy to complete the analysis up to the level of complexity they require for their needs. During user interviews, I discovered that users have different use cases for their analysis - sometimes fast, abstract themes were needed for a time-critical decision, whereas other times deep, nuanced analysis is crucial for the user.

The incumbent solution offered a one-size-fits-all solution for users, often dictating a particular step-by-step process which users described as frustrating.



Outcome

In just two weeks, I created an end-to-end interactive prototype that outperformed the existing design solution, which had been in development for over a year. User testing measured task understanding, task completion, user delight, and willingness to share contact details and intention to use the product. My end-to-end solution was successful with all users, while the existing solution was seen as a work in progress and garnered no contact details.

Lessons Learned

Following the design sprint framework allowed for the creation of a successful end-to-end prototype in just two weeks. It was essential to collaborate closely with the Product Manager and draw on our shared knowledge of non-expert users. Separating each data science concept in the product flow allowed for prioritization of optimization if it proved valuable to the user.

Conclusion

In conclusion, the redesign of the user experience for complex human-in-the-loop machine learning text analysis resulted in an intuitive experience for non-expert users. The end-to-end prototype created in the design sprint outperformed the existing solution and achieved strong user engagement. Through close collaboration and separating data science concepts in the product flow, we successfully met user needs and created a scalable product for Avalanche Insights.
