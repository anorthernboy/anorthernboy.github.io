Avalanche Insights ‘human-in-the-loop’ analysis tool redesign

Complete end-to-end overhaul of a complex product flow enabling users to extract nuanced insights from open-ended text data

Overview

Avalanche Insights is a highly successful service-based consultancy operating in the progressive politics space. Having developed a robust process for extracting key themes and insights from open-ended text data, Avalanche Insights wanted to productize their services in order to scale.

Having delivered a valuable data exploration and insights experience for users, I was recruited to facilitate a design sprint with the goal of redesigning a complex user flow allowing a non-expert user to extract themes and insights from their open-ended text data rapidly through an intuitive 'human-in-the-loop' machine learning process.


Goals

- Redesign user experience for complex 'human-in-the-loop' machine learning text analysis
- Create an intuitive experience for a non-expert user


Role

- Design Sprint Facilitator
- UI/UX Designer


Tools

Discovery

This project broadly followed the design sprint framework, additional commitments across the team dictated that the timeframe would be 2-3 weeks.


The understand phase had two key elements:

UX Audit

An in depth review of the incumbent solution and the user interviews which had been conducted, presented by the Product Manager and Product Designer responsible for this product experience. The purpose of the audit was to identify the interactions which were presenting the biggest barriers to users achieving their goals.

Many challenges were identified with the incumbent solution, and through a process of card sorting the biggest issues were grouped as below:

- A disconnect between user expectations vs. actual application behavior
- An inflexible flow which limited user actions to a linear process
- Users unable to access the information they required in the interface

Feasibility workshops

To ensure solutions were grounded in feasibility we conducted a number of lightning talks with our data science lead, engineering manager, and Avalanche Insights analysts (experts in deriving themes and insights from open-ended text data). This was an opportunity to encourage creative thinking and source ideas for utilizing the proprietary data science from all participants in the design sprint.

The workshops resulted in a shared creative space in MIRO, where how might we statements were grouped into themes using affinity mapping. The purpose of this shared space was to identify opportunities to meet user needs.


In the definition stage I facilitated a workshop with the below goals (and the overarching ambition to achieve stakeholder buy in):

- Outline a problem statement
- Review of user personas and user journeys
- Document any major assumptions relevant to the project
- Agree on testing success metrics


The problem statement which was agreed:

How can we use semantic diversity cluster analysis, natural language processing, and classification models to create an accessible (low cognitive load), usable (intuitive), and feasible product flow which allows non-expert users to extract themes from open-ended text data?


It should be noted at this stage, the problem statement was narrower than ideal as one of the major assumptions was the requirement to reuse and repackage any data science work which had already been completed - so the clustering technique and classification models could not be changed.

Given the limitations of Figma prototypes in testing complex data-driven user experiences, it was also agreed the success metrics would be grounded in user understanding (can the user explain an interface, suggest what is happening) and task completion on a series of micro-interactions which had been identified by Avalanche Insights analysts as important steps to take in extracting themes from open-ended text data.

With more resources, time - or both - the testing process could be supported by a research engineer to deploy a proof-of-concept prototype with dynamic data.

Process

Sketching was conducted asynchronously, with guidance provided on some methods for divergent thinking such as Crazy 8’s - whereby a participant tries to generate as many low fidelity ideas as possible in a short time period. Another short session was arranged to share Crazy 8’s ideas, and then each participant was asked to work on a solution sketch - an end-to-end proposal to address the problem statement.

I enlisted a decider to review the solution sketches and advise on the most promising idea to move into the testing phase.

My idea was selected as the biggest opportunity based on the idea of giving a user a headstart in their analysis and also the separation of concerns I built into the product flow - each data science concept is separated which allows each concept to be prioritized for optimization if it is discovered to be particularly valuable to the user in achieving their goals.

The next task was to gradually increase the fidelity. First going from sketch to wireframe, then higher fidelity screens and finally an interactive prototype for testing. I collaborated very closely with the Product Manager I worked with on the Avalanche Insights data exploration platform design - seeking feedback and iterating while drawing on our shared knowledge of non-expert users we had gained over the course of a huge number of discovery interviews.

Outcome

Given the requirements set at the outset - broadly, use this data science approach and include these micro-interactions - it became very apparent that there would be real value in conducting a rumble whereby my proposal would be tested directly against the existing solution, following the same interview script, tasks, and success metrics.

A number of key features are presented in the following user flows.

Start Analysis

After the user has completed their upload to the platform, they can access the analysis dashboard in an empty state. There is a call to action, leading the user to begin the machine assisted analysis process.

Cluster analysis is performed on the upload and the user is able to start their analysis with some smart themes which are fully customizable. This feature was a key value proposition discovered through user interviews where users would describe feeling overwhelmed starting with a blank slate and would like a headstart on their analysis.

In the incumbent solution, the user was confused by the requirement to confirm every AI suggestion. My solution automatically adds the AI suggestions, while making it really straightforward to manage the suggestions.



Explore Smart Themes

The user can access everything in a single interface. They can explore themes and responses; search, sort, and filter; and generally move in the direction the analysis takes them. In the UX audit of the incumbent solution, users noted how restricted they were in the direction they could take with their analysis.



Add a Response to a Theme

Associating responses with themes is the defining micro-interaction required to complete the analysis. The interaction in my solution is based on familiar industry patterns often seen in email inbox management. Clicking a response to select, the user is presented with an action menu. The user can also drag responses onto a theme.



Add Multiple Responses to a New Theme

The user can select multiple responses and access the same action menu as above - or drag the responses on to an existing theme, or the 'Create new theme' button.

The UX audit surfaced some very clumsy interactions to add multiple responses to a theme, including holding down the shift key to multi-select. Although this action can be intuitive to a product designer, target users were not able to complete the task with this interaction.



See Similar Responses

See similar responses was a novel use of the existing data science which I developed during the ideation process. All responses are plotted on a vector map during the cluster analysis, see similar allows the user to select a reference point and explore the closest responses from the vector map. In allowing the user to surface and analyze multiple similar responses, the effeciency of their process is massively increased.



Organize Themes

Almost all users identified a requirement to organize themes and create theme hierarchies. During the UX audit this feature was identified as a major weakness for the inclumbent solution, as the user was either fully or partly restricted in when they could complete this step - either at the end of the process of by navigating to a new tab.

My solution embedded the ability to organize themes at any point in the analysis, and in the interface that the user is working. Crucially, I understood that organizing themes is an iterative process, theme hierarchies are fluid and will change as the analysis proceeds - making access a key requirement for the user.



Train Theme

Training a theme allows the user to provide direct feedback to the AI classification model. This is a key feature of the platform as it enables autotagging, which can increase the efficiency of the analysis exponentially.

The incumbent solution delivered this interaction in a very busy interface, which caused confusion during user interviews. My solution utilizes the thumbs-up, thumbs-down interaction (which users liked) but reduces the complexity of the interface and thus reduces cognitive load for the user. Within the drawer in my iterface, the user can also add or remove responses which have been added by the user or added by AI - this interaction provides further feedback to the classification model without encroaching on the user's analysis flow.



Analysis Complete

The user has the autonomy to complete the analysis up to the level of complexity they require for their needs. During user interviews, I discovered that users have different use cases for their analysis - sometimes fast, abstract themes were needed for a time-critical decision, whereas other times deep, nuanced analysis is crucial for the user.

The incumbent solution offered a one-size-fits-all solution for user, often dictating a particular step-by-step process which users described as frustrating.



Key Takeaways

Following the design sprint framework, in just two weeks I created an end-to-end interactive prototype which included representations of a number of human-machine interactions.

In user testing, this experience strongly outperformed the existing design solution which had been in development for over a year.

The metrics used to measure this included task understanding, task completion, user delight, and - most importantly - whether the user was willing to share contact details and showed definite intention to use the product. On this last metric, my end-to-end solution was successful with all users, whereas the existing solution was seen as a work in progress and garnered no contact details.
