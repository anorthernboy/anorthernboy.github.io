Avalanche Insights data exploration platform design

A zero-to-one design project to enable a shift from a service-based to SaaS-based revenue model

Overview

Avalanche Insights is a highly successful service-based consultancy operating in the progressive politics space. Having developed a robust process for extracting key themes and insights from open-ended text data, Avalanche Insights wanted to productize their services in order to scale.

Starting from zero, we launched a Beta platform - a data exploration and insights experience - within six-months. The platform was supported by a magic box experience in the backend - as the proprietary data science was not sufficiently productized we had analysts using other software to derive themes and insights to test the value proposition.


Goals

- Build a platform to display themes and insights derived by analysts
- Build a platform which allows a general user to explore themes and insights derived by analysts


Role

- User Researcher
- UI/UX Designer

Discovery

The project started with an in-depth competitive analysis, covering both direct and indirect competition. From a product perspective, this stage was important in outlining the beginning of a Minimum Viable Product (MVP) feature set. From a design perspective, I was able to evaluate the design language typically utilized in this market. Scoping out product testimonials and use cases allowed me to generate some initial hypotheses on potential beachhead markets.

After creating test cards for potential users, the next stage was conducting a survey to gather some validation signals. Using a survey allowed me to very quickly get a read on the value proposition for each target user group.

The survey results suggested a number of use cases whereby users within the group met the test criteria, which included metrics such as budget for tooling, conducting surveys on a regular basis, and sharing key insights drawn from survey data. The user groups identified were:

- human resources managers conducting workplace surveys
- marketing teams gauging new product launches
- product managers or user researchers gathering product feedback

With these use cases identified, the next stage of discovery required more depth, which meant user interviews. Again, hypotheses were formed and documented in test cards. The purpose of the interviews was to further my understanding of the jobs-to-be-done and the related pain points users experience within their existing processes.

From the discovery interviews I was able to outline my user personas and begin to shape empathy maps and user journey maps for each persona. The journey maps are particularly valuable at this stage, as they can surface the most pressing pain points which the product can solve for.

I started to sketch and create product concepts around key pain points, these sketches and concepts informed further rounds of discovery interviews - gathering feedback, testing hypotheses, updating personas and journeys. Through a number of iterations, my understanding of the user and their requirements exponentially improved and I continually refined the product before moving onto the solution design.

Process

Working from the findings of the discovery iterations, I took the most promising concepts and gradually began to increase the fidelity. First going from sketch to wireframe, then higher fidelity screens and finally interactive prototypes. At each stage I sought feedback from potential users through further interviews and usability tests.

In the spirit of lean methodology, I chose to use an open source design system which included excellent documentation and best practice design patterns (in addition to the modular design components). Using an open source design system saved a huge amount of time and effort at all stages of the product development lifecycle. It ensured the design was consistent across pages, allowed me to focus on solving new or novel problems, and made the handoff process much more fluid.

In continuing the test-validate-iterate cycle employed in the discovery phase I was able to improve both the utility and efficiency of my design process and increase the likelihood of creating value for the end user.

Outcome

The discovery and design process outlined above was repeated continually to explore solutions to user problems identified and prioritized through ongoing discovery interviews. Key features supporting the value proposition are presented in the following user flows.

Upload

The onboarding flow offers signup via email of social media, with profile creation following. The user is presented with a map of Pantrii locations and asked to select a home Pantrii. Depending on revenue models there could possibly be an additional option to add multiple Pantriiâ€™s to the account as a premium feature. Users are given the option to add another user to the account and an onboarding prompt is designed to personalize or tailor content for the user.

Dashboard

Using the phone to access the Pantrii, the user receives a notification and can head straight to the menu. Adding an item is the primary CTA and there is an animated interaction on increasing the amount with the slider. When checking out, all items are dispensed from the vending machine type dispensary. Alternatively, the user might prepare an order before visiting the Pantrii and a notification on arrival asks the user if they would like to dispense.

Manage

In the interest of reducing waste, this scenario presents a smart notification prompting the user to review their previous purchases. In this flow, we are assuming Pantrii has produced smart IOT containers and is aware that the rigatoni has not yet been used. The user is taken to their Pantrii, to see all their items stored in their household. Within each item is the option to send to the marketplace (share with another user) or explore some recipe ideas.

Future Pantrii developments could move beyond store cupboard staples and provide all ingredients for a recommended recipe. Collecting behavioral data on users interacting with a given Pantrii could unlock opportunities to manage food waste across all categories.

Key Takeaways

At Avalanche Insights, I joined a company equipped to iteratively build from zero - but not without challenges, and especially ambiguity.

Within two months of joining the company, myself and my Product Manager conducted a huge amount of discovery interviews in order to establish a customer and the most pressing jobs which we could solve with the proprietary data science underpinning the product. From this we established user personas, user journeys, and prioritization matrices.

During this process I was rapidly moving between ideation and testing with interactive prototypes in order to satisfy user needs. Each feedback loop informed the user personas and user journeys which always remained our reference point.

Starting from zero, we launched a Beta platform - a data exploration and insights experience - within six-months. The platform was supported by a magic box experience in the backend - as the proprietary data science was not sufficiently productized we had analysts using other software to derive themes and insights to test the value proposition.

The platform garnered paying customers - including a global streaming platform - and received positive feedback. I continued speaking to users, preparing and testing hypotheses, and proposing features for development based on prioritization metrics.
